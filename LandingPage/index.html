Mac dictation, through the side door.

Hold a hotkey, speak, release. ChatFlow streams your mic to OpenAI Realtime, gets the transcript back, and pastes it into whatever app is focused.

The juggadu: it matches the Codex CLI OAuth PKCE flow, stores the ChatGPT token in Keychain, and talks to Realtime without an API key or billing page.

the trick

Three steps. One weirdly useful auth path.

This is the part developers care about. No vague magic, just the trail of crumbs.

01 OAuth PKCE

Open the same door Codex uses.

ChatFlow builds the Auth0/OpenAI authorize URL with the Codex CLI client ID, S256 verifier, local callback server, and Codex-specific params.

/oauth/authorize → localhost:1455/auth/callback
02 Token exchange

Swap the code for a token.

The callback code becomes a ChatGPT access token via form-urlencoded `/oauth/token`. Refresh tokens live in Keychain, where they belong.

grant_type=authorization_code
03 Realtime dictation

Stream audio, receive text.

Core Audio feeds PCM16 chunks to the Realtime WebSocket. The server sends transcription deltas and a final transcript, then ChatFlow pastes it.

input_audio_buffer.append

why this is interesting

It feels like a product because the hack is doing real work.

No API key

Use your existing ChatGPT account. No API dashboard ceremony.

Native macOS loop

Global hotkey, menu bar UI, permissions onboarding, and focused-app paste.

Realtime by default

Pre-connects when it can, streams chunks as you speak, and waits for deltas.

Works until it doesn’t

Honest shelf-life note. Auth paths move. Duct tape has a calendar.

quick start

Clone, build, speak.

First launch asks for the boring macOS things. Microphone for audio, Accessibility/Input Monitoring for hotkeys and text injection, and ChatGPT sign-in for the token.
